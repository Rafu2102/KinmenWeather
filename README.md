## 📂 專案報告大綱 (書面與 PPT 參考內容)

### 1. 研究動機與目的
* **研究背景**：金門氣候受季風影響顯著，且地理位置接近福建沿海，春季常有大霧阻斷航運，冬季東北季風寒冷強勁。為進行科學化的氣象分析，我們決定以真實觀測數據建立 AI 模型。
* **研究目的**：本專案完全摒棄模擬與猜測成分，直接讀取中央氣象署的官方每日觀測檔（例如測站代號 `467110` 觀測資料），對多達 34 個數值型氣候變數（包含氣壓、氣溫、濕度、風速、風向、降雨量、蒸發量、紫外線指數與土壤地溫等）進行分析，並訓練前沿的機器學習集成模型來預測今日的所有氣象觀測值，為氣象分析提供客觀的模型參考。

### 2. 資料集特徵與高階特徵工程
* **資料來源**：中央氣象署下載之金門測站（`467110`）官方每日觀測 CSV 檔（放於 [日月CSV](file:///d:/AI%20%E4%BA%BA%E5%B7%A5%E6%99%BA%E6%85%A7%E5%B0%8E%E8%AB%96/%E9%87%91%E9%96%80%E5%A4%A9%E6%B0%A3%E9%A0%90%E6%B8%AC%E5%B0%88%E6%A1%88/%E6%97%A5%E6%9C%88CSV) 目錄中）。
* **格式標準化與清理**：
  * 跳過氣象局原始檔案的第一列中文說明，以第二列英文縮寫（`ObsTime`, `Temperature`, `RH`, `WS`, `Precp` 等）為 DataFrame 欄位名。
  * 清理缺失值：將無觀測資料的 `--` 取代為 `NaN`，並以鄰近日前後填充（Forward Fill / Backward Fill）修補。
  * 清理微量降雨：將降雨量欄位 `Precp` 中代表微量雨量的 `T` 值置換為 `0.0`，並全部轉換為浮點數型態。
  * 日期生成：由檔名解析出年份與月份（如 `467110-2026-06.csv` 得到 `2026` 年 `6` 月），並與觀測日 `ObsTime` 組合成標準的 `Date` 欄位。
* **高階時間特徵週期性編碼 (Cyclical Encoding)**：
  氣候具有強烈的季節週期性。我們對月份（Month）和一年中的第幾天（DayOfYear）進行正弦與餘弦轉換（`sin` 與 `cos` 轉換），將時間變數映射到二維圓形空間。這可確保 12 月與 1 月、365 天與 1 天在數學上保持鄰近關係，大幅提升時間序列預測精度。
* **一階動能差值特徵 (First-Order Difference)**：
  除了昨日天氣數值（Lag 1）與前日天氣數值（Lag 2）外，我們計算了 `diff1 = lag1 - lag2`。這代表氣溫、氣壓或濕度昨日相對於前日的「變化速率與動能趨勢」（例如氣壓急升或急降），為模型提供了捕捉動態氣候變遷的特徵。

### 3. 前沿機器學習預測演算法
為了取得優良的預測成效，我們實作了三種不同數學機制的基底模型，並採取 Blending 加權集成：
* **嶺回歸 (Ridge Regression)**：
  * 原理：在常規最小平方法中加入 L2 正規化（Ridge 懲罰項）。
  * 作用：當氣候滯後特徵高度共線（例如昨天的氣溫與前天的氣溫極度相關）時，Ridge 能有效控制參數大小，防止模型過擬合，並作為穩健的線性 Baseline。
* **隨機森林回歸 (Random Forest Regressor)**：
  * 原理：基於自助取樣（Bootstrap）和隨機特徵分裂的袋裝集成學習（Bagging）。
  * 作用：對極端天氣和噪聲有極佳的魯棒性，並可輸出非線性的「特徵重要性權重」進行可解釋性分析。
* **梯度提升決策樹 (HistGradientBoostingRegressor)**：
  * 原理：基於直方圖最佳化的高效梯度提升演算法（GBDT），類似於 LightGBM，每次迭代皆在擬合前一次預測的殘差。
  * 作用：目前表格式資料（Tabular Data）中性能優異的預測模型，能精準捕捉非線性且高維度交叉的複雜天氣交互關係。
* ** blending 科學加權集成模型**：
  * 結合公式：`0.1 * Ridge + 0.4 * RandomForest + 0.5 * HistGradientBoosting`
  * 作用：融合了線性泛化性、Bagging 穩定性與 Boosting 的高擬合能力，形成預測偏差與變異數最小的綜合預報模型。

### 4. 資料科學圖表解讀 (報告視覺化重點)
本系統提供全方位、動態的科學圖表分析，您可直接將其作為期慢報告的亮點：

#### 📊 歷史數據探索圖表
* **趨勢分析折線圖 (加 30 日滾動平均)**：顯示氣候變數的長期走向，滾動平均線可平滑每日的高頻噪音，展示季節或年度的核心變化規律。
* **數據分佈直方圖與 KDE 密度曲線**：直觀呈現變數的分佈形態（如氣溫呈現雙峰分佈，降雨量呈現極端右偏分佈），並疊加平滑的核密度估計，驗證變數的統計特性。
* **月份季節性箱線圖 (Boxplot)**：將十年數據按月份分組繪製。箱體展示各月的中位數、四分位距以及極端異常點，能一眼看出金門的冬寒夏熱與春秋氣候過渡期。
* **關鍵氣候指標相關性熱力圖 (Correlation Heatmap)**：展示各欄位之間的相互關係。例如「氣溫」與「地溫」強正相關，「氣壓」與「氣溫」顯著負相關，為氣候學分析提供科學佐證。

#### ⚙️ 機器學習診斷與成效圖表 (支援 34 個觀測變數動態渲染)
* **特徵重要性分析圖 (Feature Importance)**：揭示昨日各指標中，哪些對今日預測貢獻最大（如昨日均溫通常為預測今日溫度的最關鍵因子，其次為週期時間特徵）。
* **實際與預測對比折線圖**：展示最近 90 天內，組合集成模型預估值與氣象署真實觀測值的追隨擬合度，展現模型預報的精確軌跡。
* **預測與實際擬合散佈圖 (Scatter Fit)**：繪製對角完美預測線，散佈點越密集分佈於對角線周圍，代表模型預測精度越出色。
* **預測殘差時序圖 (Residual Plot)**：繪製每日預測誤差（預測 - 實際）。殘差應呈隨機白噪音散佈於 0 線上下，無明顯季節性規律，證明模型已完整捕捉有用資訊。
* **誤差分佈直方圖與常態檢驗 (Error Distribution)**：將預測誤差繪製直方圖並進行常態分佈擬合。當誤差符合均值為 0 的常態分佈時，在數學上證明了該模型是無偏估計，且預測誤差僅來自於不可避免的隨機噪聲。

## ⚡ 專案快速執行指南

### 步驟 1：安裝依賴套件
在專案根目錄下，開啟終端機 (PowerShell 或 Command Prompt) 執行以下指令：
```bash
pip install -r requirements.txt
```

### 步驟 2：解析 CSV 並訓練模型
您只需在 `日月CSV/` 目錄中丟入氣象署下載的 CSV 檔案，接著單獨執行 [model.py](file:///d:/AI%20%E4%BA%BA%E5%B7%A5%E6%99%BA%E6%85%A7%E5%B0%8E%E8%AB%96/%E9%87%91%E9%96%80%E5%A4%A9%E6%B0%A3%E9%A0%90%E6%B8%AC%E5%B0%88%E6%A1%88/model.py)：
```bash
python model.py
```
這會自動讀取、合併並處理真實觀測資料，並在 [plots/](file:///d:/AI%20%E4%BA%BA%E5%B7%A5%E6%99%BA%E6%85%A7%E5%B0%8E%E8%AB%96/%E9%87%91%E9%96%80%E5%A4%A9%E6%B0%A3%E9%A0%90%E6%B8%AC%E5%B0%88%E6%A1%88/plots) 目錄下產生氣溫預測的基準圖表，並完成全部 34 個氣象觀測欄位的 AI 訓練與評估。

### 步驟 3：啟動 Streamlit 網頁儀表板
執行以下指令啟動網頁服務：
```bash
streamlit run app.py
```
執行後，網頁將自動開啟 `http://localhost:8501`，即可在網頁端隨選 34 個氣候指標，即時互動生成上述所有精美圖表！
